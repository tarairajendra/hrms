
# HRMS

## Requerements

1. Spend Tracker
    - user can record their spendings
    - the spendings needs to be categorized (e.g. "meals", "office_stationaries", "electricity")
    - spendings can split among users
    - it needs to be approved/rejected by moderator (remark needed if rejected)
    - after rejection spender needs to edit and resubmit for approval
    - user can see their spending at any moment

2. Track attendance

3. Leave Application
    - user able to apply for leaves
    - the application should have reason_of_leave, from_date, to_date, timestamp
    - the application should accepted/rejected by their manager
